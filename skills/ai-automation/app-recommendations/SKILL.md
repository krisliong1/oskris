# App Recommendations Skill

## 🎯 核心原则

**高信息密度 = 省 90-95% Token**:
- 保留**所有重点信息**
- 删除**废话、重复、冗长解释**
- 结构化呈现(表格优于段落)
- 一句话说清楚,绝不用三句

**什么是废话**:
- ❌ "这是一个非常好用的..." (啰嗦的修饰词)
- ❌ "让我为你介绍..." (多余的开场白)
- ❌ "你可以用它来..." (重复的用途说明)
- ❌ 背景故事、公司历史 (用户不关心)

**什么是重点**:
- ✅ 准确的 App 名字、价格、平台
- ✅ 核心功能(3-4条足够)
- ✅ 适用场景(帮决策)
- ✅ 对比时的差异点

---

## 📱 触发条件

当用户询问:
- "推荐 XX 类型的 App"
- "有什么好用的 XX App"
- "XX App 叫什么名字"
- "在 App Store 搜什么"
- "对比 XX 和 YY"

---

## 🎛️ 响应原则

**平衡信息量和 token 消耗**:
- 提供足够信息让用户做决策
- 避免冗长重复的内容
- 结构化输出提高可读性
- 用户可要求更多细节

---

## 📋 标准响应模板

### **单个 App 推荐**

**高效格式**(紧凑但完整):
```
[App Store 名字] | [开发商]
平台: [iOS/Android/Web] | 价格: $X/月

核心功能:
• [功能1]
• [功能2]  
• [功能3]

适合: [使用场景1句话]
```

**Token 估算**: 80-120 tokens

**对比之前**(冗长版 ~500 tokens):
```
❌ 废话版:
"[App名]是一个非常优秀的应用程序,
由知名公司[开发商]开发。这个应用
提供了丰富的功能,包括[功能1],
让用户可以...(一堆解释)

您可以在iOS和Android平台上下载
使用这个应用。定价为每月$X,
提供了非常好的性价比..."

vs

✅ 高密度版:
"[App名] | [开发商]
iOS/Android | $X/月
• [功能1]
• [功能2]
适合: [场景]"
```

---

### **多个 App 对比(2-5个)**

**高效格式**(表格 + 差异点):
```markdown
| App | 平台 | 价格 | 核心特点 |
|-----|------|------|----------|
| [名1] | iOS/Android | $X | [1句话优势] |
| [名2] | iOS/Android | $Y | [1句话优势] |
| [名3] | Web/Mobile | 免费 | [1句话优势] |

关键差异:
• [差异点1]
• [差异点2]

选择: [需求A]→[App1] | [需求B]→[App2]
```

**Token 估算**: 120-200 tokens (3个App)

**对比之前**(重复结构 ~800 tokens):
```
❌ 重复版:
### 1. [App1]
平台: iOS/Android
价格: $X
特点: [一段话]
优势: [一段话]
劣势: [一段话]

### 2. [App2]  
平台: iOS/Android
价格: $Y
特点: [一段话]
优势: [一段话]
劣势: [一段话]

(每个App都重复相同结构)

vs

✅ 表格版:
| App | 平台 | 价格 | 优势 |
直接对比,一目了然!
```

**省 token 秘诀**:
- 表格 > 重复段落 (省 70%)
- 只列差异点 > 每个都写优缺点 (省 60%)
- 1句话 > 1段话 (省 80%)

**避免**:
- ❌ 长篇说明
- ❌ 重复功能列表
- ❌ 过多示例
- ❌ 无关背景信息

## ✂️ 去废话核心技巧

### **技巧 1: 表格化(省 80%)**
```
❌ 段落:
"Poe支持iOS和Android平台。价格是每月$20。
它的特点是可以访问多个AI模型..."

✅ 表格:
| Poe | iOS/Android | $20/月 | 多AI |
```

### **技巧 2: 只写差异(省 73%)**
```
❌ 全写每个优缺点

✅ 只写差异:
"Poe多AI有限制 | Claude单AI功能全"
```

### **技巧 3: 删修饰词(省 85%)**
```
❌ "这是一个非常优秀的、功能强大的应用"
✅ "多AI聚合平台"
```

### **技巧 4: 合并重复(省 80%)**
```
❌ 功能1: XX | 功能2: YY | 功能3: ZZ
✅ 功能: XX、YY、ZZ
```

### **技巧 5: 去背景故事(省 92%)**
```
❌ "Poe由Quora开发,Quora是2009年成立的..."
✅ "Poe | Quora出品"
```

**避免**:
- ❌ 长篇说明
- ❌ 重复功能列表
- ❌ 过多示例
- ❌ 无关背景信息

## 🔍 搜索验证规则

### **必须验证**:
- App Store 真实名字
- 开发商
- 平台可用性(iOS/Android/Web)
- 价格(免费/付费)

### **搜索流程**:
```
1. web_search: "[App名] App Store iOS"
2. 确认官方名字
3. 记录开发商
4. 检查 Android 版本
5. 提供精简信息
```

### **不存在的 App**:
```
❌ "Chatbot Arena 没有 App"

明确说明:
• 只有网页版
• 提供网址
• 可选:添加到主屏幕方法(如用户需要)
```

## 📌 总结

### **核心策略**: 保留重点 + 删废话 = 省 90-95%

1. **用表格代替段落** - 省 70-80%
2. **只写差异不重复** - 省 60-70%  
3. **删所有修饰词** - 省 80-85%
4. **合并重复结构** - 省 75-80%
5. **去背景故事** - 省 90-95%

### **Token 对比**:

```
之前(冗长版):
单App: 500 tokens
3个对比: 2,000 tokens
深度: 3,000+ tokens

现在(高密度版):
单App: 20-80 tokens (省 84-96%)
3个对比: 150-200 tokens (省 90-92.5%)
深度: 200-300 tokens (省 90-93%)

平均节省: 90-95%!
```

✅ **所有重点信息都保留**
- App名字、价格、平台 ✓
- 核心功能(3-4条) ✓
- 适用场景 ✓
- 选择建议 ✓

✅ **删除的都是废话**
