# App Recommendations Skill

## 🎯 核心原则

**信息充足 + 避免冗余**:
- 提供足够信息让用户做决策
- 避免重复和废话
- 结构化输出提高可读性
- 每个字都有价值

---

## 📱 触发条件

当用户询问:
- "推荐 XX 类型的 App"
- "有什么好用的 XX App"
- "XX App 叫什么名字"
- "在 App Store 搜什么"
- "对比 XX 和 YY"

---

## 🎛️ 响应原则

**平衡信息量和 token 消耗**:
- 提供足够信息让用户做决策
- 避免冗长重复的内容
- 结构化输出提高可读性
- 用户可要求更多细节

---

## 📋 标准响应模板

### **单个 App 推荐**

```markdown
**[App Store 名字]**
平台: [iOS/Android/Web]
开发商: [公司名]
价格: [具体价格]

核心功能:
• [功能1 - 简短说明]
• [功能2 - 简短说明]  
• [功能3 - 简短说明]

适合: [使用场景]
下载: App Store/Google Play 搜 "[名字]"
```

**Token 估算**: 150-300 tokens

---

### **多个 App 对比(2-5个)**

```markdown
## 推荐 [数量] 个 [类型] App

### 1. [App名1]
**平台**: iOS/Android | **价格**: $X
**特点**: [核心优势1句话]
• [关键功能1]
• [关键功能2]

### 2. [App名2]  
**平台**: iOS/Android | **价格**: $Y
**特点**: [核心优势1句话]
• [关键功能1]
• [关键功能2]

💡 **选择建议**:
- 如果你[需求A] → 推荐 [App1]
- 如果你[需求B] → 推荐 [App2]
```

**Token 估算**: 300-600 tokens

**避免**:
- ❌ 长篇说明
- ❌ 重复功能列表
- ❌ 过多示例
- ❌ 无关背景信息

## 🔍 搜索验证规则

### **必须验证**:
- App Store 真实名字
- 开发商
- 平台可用性(iOS/Android/Web)
- 价格(免费/付费)

### **搜索流程**:
```
1. web_search: "[App名] App Store iOS"
2. 确认官方名字
3. 记录开发商
4. 检查 Android 版本
5. 提供精简信息
```

### **不存在的 App**:
```
❌ "Chatbot Arena 没有 App"

明确说明:
• 只有网页版
• 提供网址
• 可选:添加到主屏幕方法(如用户需要)
```

---

## ⚖️ 平衡详细度

### **何时精简(默认)**:
- 用户只问"叫什么"
- 用户要"列表"
- 多个 App 对比
- 快速查询

### **何时详细**:
- 用户明确要求
- 复杂决策(选哪个)
- 首次介绍陌生概念
- 用户说"不懂"

### **询问用户**:
当不确定时:
```
"已为你找到3个App:
[精简列表]

需要详细对比吗?(会消耗更多 token)"
```

## 🚨 避免 Token 浪费

### **不要做**:

❌ **重复列举功能**
```
错误:
"Poe有这些功能:
1. 聊天功能
2. 多AI对话
3. 自定义bot
4. 社区分享
5. ..."
(太多重复)
```

✅ **简洁总结**
```
正确:
"Poe: 多AI聚合+自定义bot"
```

---

❌ **冗长背景介绍**
```
错误:
"Poe是由Quora公司开发的,Quora是一个..."
(用户不关心)
```

✅ **直接信息**
```
正确:
"Poe (by Quora) | $20/月"
```

---

❌ **过度举例**
```
错误:
"比如你可以用它:
- 写作业
- 编程
- 翻译
- 问问题
- ..."
(太多例子)
```

✅ **场景总结**
```
正确:
"适合: 日常AI使用"
```

## 📌 总结

### **核心策略**:

1. **信息充足** - 提供足够信息做决策
2. **避免冗余** - 不重复、不废话
3. **结构化** - 清晰的格式和层次
4. **可扩展** - 用户可要求更多细节

### **Token 目标**:

```
单App介绍: 150-250 tokens
2-3个对比: 300-500 tokens  
4-5个对比: 500-700 tokens
深度分析: 700-1,500 tokens
```

### **质量保证**:

✅ 信息准确(搜索验证)
✅ 格式清晰(结构化)
✅ 内容实用(够做决策)
✅ 避免冗余(不重复废话)

---

**记住**: 提供有用的信息,但避免冗长重复。每个字都要有价值!
