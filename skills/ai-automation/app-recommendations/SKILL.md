# App Recommendations Skill

## 🎯 核心原则

**Token 效率优先**:
- 默认精简模式(省 token)
- 用户可选详细模式
- 避免重复内容
- 聚焦核心信息

---

## 📱 触发条件

当用户询问:
- "推荐 XX 类型的 App"
- "有什么好用的 XX App"
- "XX App 叫什么名字"
- "在 App Store 搜什么"
- "对比 XX 和 YY"

---

## 🎛️ 两种响应模式

### **模式 1: 精简模式(默认)**

**输出格式**:
```
App Store 名字 | 平台 | 价格
━━━━━━━━━━━━━━━━━━━━━━━━
简短说明(1行)
核心特点(最多3条)
```

**Token 估算**: 200-500 tokens

**示例**:
```
Poe – Fast AI Chat | iOS/Android | 部分免费
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
多AI聚合平台,包含Claude/GPT/Gemini
• 一个App用多个AI
• 100万+自定义bot
• $20/月全包
```

---

### **模式 2: 详细模式(用户要求)**

触发词:
- "详细介绍"
- "说清楚一点"
- "完整对比"
- "更多信息"

**输出格式**:
```
完整 App Store 名字
开发商
平台可用性
定价方案
详细功能列表
优缺点对比
使用场景建议
```

**Token 估算**: 1,000-3,000 tokens

---

## 📋 标准响应模板

### **单个 App 推荐**

```markdown
**[App Store 名字]**
平台: [iOS/Android/Web]
价格: [免费/付费/订阅]

核心功能:
• [功能1]
• [功能2]
• [功能3]

适合: [使用场景]
```

---

### **多个 App 对比(最多5个)**

```markdown
## 📊 快速对比

| App | 平台 | 价格 | 核心特点 |
|-----|------|------|----------|
| [名字1] | iOS/Android | $X | [1句话] |
| [名字2] | iOS/Android | $Y | [1句话] |

💡 **建议**: [根据用户需求推荐1个]
```

**避免**:
- ❌ 长篇说明
- ❌ 重复功能列表
- ❌ 过多示例
- ❌ 无关背景信息

---

## 🔍 搜索验证规则

### **必须验证**:
- App Store 真实名字
- 开发商
- 平台可用性(iOS/Android/Web)
- 价格(免费/付费)

### **搜索流程**:
```
1. web_search: "[App名] App Store iOS"
2. 确认官方名字
3. 记录开发商
4. 检查 Android 版本
5. 提供精简信息
```

### **不存在的 App**:
```
❌ "Chatbot Arena 没有 App"

明确说明:
• 只有网页版
• 提供网址
• 可选:添加到主屏幕方法(如用户需要)
```

---

## ⚖️ 平衡详细度

### **何时精简(默认)**:
- 用户只问"叫什么"
- 用户要"列表"
- 多个 App 对比
- 快速查询

### **何时详细**:
- 用户明确要求
- 复杂决策(选哪个)
- 首次介绍陌生概念
- 用户说"不懂"

### **询问用户**:
当不确定时:
```
"已为你找到3个App:
[精简列表]

需要详细对比吗?(会消耗更多 token)"
```

## 📊 Token 消耗指南

### **精简回复示例**

**单App**: 50-100 tokens
```
Poe – Fast AI Chat
iOS/Android | $20/月
多AI聚合,100万+bot
```

**3个App对比**: 150-300 tokens
```
| App | 特点 |
|-----|------|
| Poe | 多AI |
| Claude | 功能全 |
| Perplexity | 搜索强 |
```

**5个App对比**: 300-500 tokens

---

### **详细回复示例**

**单App深度**: 500-1,000 tokens
**3个App详细对比**: 1,500-2,500 tokens
**完整分析**: 2,500-5,000 tokens

## 🚨 避免 Token 浪费

### **不要做**:

❌ **重复列举功能**
```
错误:
"Poe有这些功能:
1. 聊天功能
2. 多AI对话
3. 自定义bot
4. 社区分享
5. ..."
(太多重复)
```

✅ **简洁总结**
```
正确:
"Poe: 多AI聚合+自定义bot"
```

---

❌ **冗长背景介绍**
```
错误:
"Poe是由Quora公司开发的,Quora是一个..."
(用户不关心)
```

✅ **直接信息**
```
正确:
"Poe (by Quora) | $20/月"
```

---

❌ **过度举例**
```
错误:
"比如你可以用它:
- 写作业
- 编程
- 翻译
- 问问题
- ..."
(太多例子)
```

✅ **场景总结**
```
正确:
"适合: 日常AI使用"
```

## 💡 用户控制

### **用户可以选择**:

1. **精简模式(默认)**
   ```
   "给我推荐XX App"
   → 精简列表
   ```

2. **详细模式**
   ```
   "详细介绍XX"
   "完整对比XX和YY"
   → 详细分析
   ```

3. **指定数量**
   ```
   "推荐3个"
   → 最多3个
   ```

4. **追问**
   ```
   用户:"还有吗?"
   → 补充更多,但仍保持精简
   ```

## 📌 总结

### **核心策略**:

1. **默认精简** - 节省 token
2. **用户控制** - 想要详细时再给
3. **结构化** - 表格、列表清晰
4. **避免废话** - 直达重点
5. **智能询问** - 不确定时问用户

### **Token 目标**:

```
精简推荐: 100-300 tokens
标准对比: 300-500 tokens  
详细分析: 500-1,000 tokens
完整深度: 1,000-3,000 tokens
```

### **质量保证**:

✅ 信息准确(搜索验证)
✅ 格式清晰(结构化)
✅ 响应快速(精简优先)
✅ 用户控制(可选详细)

---

**记住**: Pro plan 有配额限制,每次对话要考虑 token 消耗。精简不代表质量低,而是信息密度高!
