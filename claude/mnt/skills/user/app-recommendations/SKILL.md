# App Recommendations Skill

## 🎯 核心原则

**详细但不废话 = 高效传达信息**:

### **必须包含的信息**:
✅ **使用场景** - 适合做什么,强项是什么
✅ **不足之处** - 不能做什么,限制在哪
✅ **替代方案** - 没App→网页+添加主屏幕
✅ **特殊用法** - 快捷指令/DMG/插件等
✅ **详细说明** - 让用户快速上手

### **什么是废话(要删)**:
❌ 空洞修饰词 - "非常优秀的"、"功能强大的"
❌ 废话开场 - "让我为您介绍"、"接下来"
❌ 冗长背景 - "由XX公司创立于20XX年..."
❌ 重复解释 - 说过的话换个方式再说一遍

### **什么不是废话(必说)**:
✅ "适合做图片/不适合编程" - 使用场景
✅ "没App但可以添加到主屏幕" - 替代方案  
✅ "Mac可用DMG/快捷指令" - 特殊用法
✅ "强项是X/不足是Y" - 帮助决策

## 📱 触发条件

当用户询问:
- "推荐 XX 类型的 App"
- "有什么好用的 XX App"
- "XX App 叫什么名字"
- "在 App Store 搜什么"
- "对比 XX 和 YY"

## 🎛️ 响应原则

**平衡信息量和 token 消耗**:
- 提供足够信息让用户做决策
- 避免冗长重复的内容
- 结构化输出提高可读性
- 用户可要求更多细节

---

## 📋 标准响应模板

### **单个 App 推荐**

**完整信息格式**:
```
[App名] | [开发商]
平台: [iOS/Android/Web/Mac等] | 价格: $X

强项:
• [擅长1]
• [擅长2]

不足:
• [限制1]
• [限制2]

适合: [具体使用场景]
不适合: [不擅长的场景]
```

**如果没有 App,必须说明替代方案**:
```
[App名] | 只有网页版
网址: [URL]

没App但可以:
• 手机浏览器访问
• 添加到主屏幕(iOS/Android操作方法)
• 快捷指令调用(如果支持)
• Mac用户: DMG/PWA等方式(如果有)

适合: [场景]
```

**Token 估算**: 150-250 tokens

---

### **多个 App 对比(2-5个)**

**高效格式**(表格 + 差异点):
```markdown
| App | 平台 | 价格 | 核心特点 |
|-----|------|------|----------|
| [名1] | iOS/Android | $X | [1句话优势] |
| [名2] | iOS/Android | $Y | [1句话优势] |
| [名3] | Web/Mobile | 免费 | [1句话优势] |

关键差异:
• [差异点1]
• [差异点2]

选择: [需求A]→[App1] | [需求B]→[App2]
```

**Token 估算**: 120-200 tokens (3个App)

**对比之前**(重复结构 ~800 tokens):
```
❌ 重复版:
### 1. [App1]
平台: iOS/Android
价格: $X
特点: [一段话]
优势: [一段话]
劣势: [一段话]

### 2. [App2]  
平台: iOS/Android
价格: $Y
特点: [一段话]
优势: [一段话]
劣势: [一段话]

(每个App都重复相同结构)

vs

✅ 表格版:
| App | 平台 | 价格 | 优势 |
直接对比,一目了然!
```

**省 token 秘诀**:
- 表格 > 重复段落 (省 70%)
- 只列差异点 > 每个都写优缺点 (省 60%)
- 1句话 > 1段话 (省 80%)

**避免**:
- ❌ 长篇说明
- ❌ 重复功能列表
- ❌ 过多示例
- ❌ 无关背景信息

---

## ✂️ 去废话核心技巧

### **技巧 1: 表格化(省 80%)**
```
❌ 段落:
"Poe支持iOS和Android平台。价格是每月$20。
它的特点是可以访问多个AI模型..."

✅ 表格:
| Poe | iOS/Android | $20/月 | 多AI |
```

### **技巧 2: 只写差异(省 73%)**
```
❌ 全写每个优缺点

✅ 只写差异:
"Poe多AI有限制 | Claude单AI功能全"
```

### **技巧 3: 删修饰词(省 85%)**
```
❌ "这是一个非常优秀的、功能强大的应用"
✅ "多AI聚合平台"
```

### **技巧 4: 合并重复(省 80%)**
```
❌ 功能1: XX | 功能2: YY | 功能3: ZZ
✅ 功能: XX、YY、ZZ
```

### **技巧 5: 去背景故事(省 92%)**
```
❌ "Poe由Quora开发,Quora是2009年成立的..."
✅ "Poe | Quora出品"
```

**避免**:
- ❌ 长篇说明
- ❌ 重复功能列表
- ❌ 过多示例
- ❌ 无关背景信息

## 🔍 搜索验证规则

### **必须验证**:
- App Store 真实名字
- 开发商
- 平台可用性(iOS/Android/Web/Mac/Windows)
- 价格(免费/付费/订阅)
- **使用场景和强项**
- **不足和限制**

### **没有 App 时必须提供**:
- 网页版网址
- 添加到主屏幕方法(iOS/Android)
- 快捷指令调用方式(如果支持)
- Mac替代方案(DMG/PWA/浏览器插件等)
- Windows替代方案(如果有)

### **搜索流程**:
```
1. web_search: "[App名] App Store iOS Android"
2. 确认官方名字和平台
3. 如果没App:
   - 搜索网页版网址
   - 搜索"[App名] 快捷指令"
   - 搜索"[App名] Mac 使用方法"
4. 记录使用场景和限制
5. 提供完整信息
```

### **搜索流程**:
```
1. web_search: "[App名] App Store iOS"
2. 确认官方名字
3. 记录开发商
4. 检查 Android 版本
5. 提供精简信息
```

### **不存在的 App 怎么处理**:
```
✅ 必须详细说明:

[App名] | 只有网页版
网址: [URL]
功能: [做什么用的]

没App的替代方案:
• 网页版: [URL]
• 添加到主屏幕(当作App用):
  - iOS: Safari打开网址→分享→添加到主屏幕
  - Android: Chrome打开网址→菜单→添加到主屏幕
• 快捷指令: [如果支持,提供方法]
• Mac用户: [DMG/浏览器/PWA等]

强项: [擅长什么]
不足: [相比原生App的限制]
适合: [使用场景]
```

## 📌 总结

### **核心原则**: 详细 + 实用 + 不废话

1. **必须说明使用场景** - 强项/不足/适合/不适合
2. **必须提供替代方案** - 没App→网页+主屏幕+快捷指令
3. **必须帮助决策** - 清楚说明选哪个
4. **删除空洞修饰** - 不说"优秀""强大"等废话
5. **结构化呈现** - 表格/列表清晰

### **详细但不废话的体现**:

✅ **详细(必说)**:
- 使用场景: 适合做图/不适合编程
- 强项不足: 擅长X/限制Y
- 替代方案: 网页+主屏幕步骤
- 特殊用法: 快捷指令/DMG/插件
- 选择建议: X需求→A | Y需求→B

❌ **废话(要删)**:
- "这是一个非常优秀的应用"
- "由知名公司开发"
- "让我为您介绍"
- 公司背景故事

### **实际对比**:

```
✅ 详细实用版(300 tokens):
- 说明没App但有网页版

- 说明强项(免费/对比AI)

- 说明适合/不适合场景
```
